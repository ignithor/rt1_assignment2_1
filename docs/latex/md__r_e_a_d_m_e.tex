\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{Summary}\label{md__r_e_a_d_m_e_autotoc_md1}
This package demonstrates the implementation of a task using an action client in R\+OS. The goal is to manage robot navigation to a target while ensuring non-\/blocking behavior. The solution is developed across two nodes and a launch file, as described below\+:\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsection{Nodes Implemented\+:}\label{md__r_e_a_d_m_e_autotoc_md2}
{\itshape Action Client Node and subscriber-\/publisher\+:}


\begin{DoxyItemize}
\item Sends target coordinates (x, y) to the action server.
\item Provides the ability to cancel the target during execution.
\item Monitors feedback and status from the action server to determine when the target is reached.
\item Publishes robot position and velocity as a custom message (x, y, vel\+\_\+x, vel\+\_\+z) based on data from the /odom topic.
\end{DoxyItemize}

{\itshape Service Node\+:}


\begin{DoxyItemize}
\item Responds to requests with the coordinates of the last target sent by the user.
\end{DoxyItemize}\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{Launch File\+:}\label{md__r_e_a_d_m_e_autotoc_md3}
